# 文件搜索工具 - 启动说明

## ⚠️ 重要提示

**必须同时运行两个服务器：**
1. **后端服务器** - 负责搜索 C 盘和 D 盘的文件
2. **前端服务器** - 提供用户界面

---

## 🚀 方法一：一键启动（推荐）

在项目根目录运行：

```bash
cd d:\SoloApp
node start-file-search.js
```

这会自动启动两个服务器。

---

## 🛠️ 方法二：分步启动

### 步骤 1：启动后端服务器

打开**第一个终端**，运行：

```bash
cd d:\SoloApp
npm run server:dev
```

你应该看到：
```
============================================================
✅ 文件搜索服务器启动成功！
📡 监听端口: http://localhost:3001
============================================================
```

**保持这个终端打开！**

### 步骤 2：启动前端服务器

打开**第二个终端**，运行：

```bash
cd d:\SoloApp
npm run dev
```

你应该看到：
```
VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
```

### 步骤 3：访问工具

在浏览器中打开：
```
http://localhost:5173/file-search
```

---

## 🔍 如何判断是否成功

### 1. 看页面右上角的模式标签：
- ✅ **🌐 服务器模式** - 后端正常运行，可以搜索真实文件
- ❌ **📝 演示模式** - 后端未启动，只能看到示例

### 2. 看浏览器控制台（F12）：
- 如果显示 `Server error: ERR_ABORTED` → 后端服务器未启动

### 3. 看后端服务器终端：
- 如果有 `收到搜索请求` 日志 → 说明连接成功

---

## ❌ 如果遇到问题

### 问题 1：端口被占用

如果提示 `Port 3001 is already in use` 或 `Port 5173 is already in use`：

```bash
# Windows
netstat -ano | findstr :3001
taskkill /PID <进程ID> /F

# 或者使用其他端口
```

### 问题 2：找不到 npm 命令

确保已安装 Node.js：
```bash
node --version  # 应该显示 v18.x 或更高
npm --version   # 应该显示 9.x 或更高
```

### 问题 3：npm install 失败

```bash
cd d:\SoloApp
rm -rf node_modules package-lock.json
npm install
```

### 问题 4：tsx 命令找不到

```bash
npm install tsx --save-dev
```

## 📊 测试连接

如果不确定后端是否运行，可以测试：

### 方法 1：使用浏览器
打开浏览器访问：
```
http://localhost:3001/api/health
```

如果显示 JSON 数据 `{"status":"ok",...}` → 后端运行正常

### 方法 2：使用 curl（如果有安装）
```bash
curl http://localhost:3001/api/health
```

## 🎯 搜索功能使用

### 1. 搜索所有文件（C 盘和 D 盘）
- 直接点击"搜索"按钮（不填关键词，不选类型）

### 2. 按文件名搜索
- 输入关键词（如：`document`）
- 选择"按文件名"
- 点击"搜索"

### 3. 按文件类型搜索
- 选择文件类型（如：`.docx`, `.pdf`）
- 选择"按后缀名"
- 点击"搜索"

### 4. 组合搜索
- 输入关键词 + 选择文件类型
- 点击"搜索"

---

## 📝 注意事项

1. **首次搜索可能较慢** - 需要扫描整个 C 盘和 D 盘
2. **搜索结果限制** - 最多返回 1000 个结果
3. **搜索深度** - 最多深入 8 层目录
4. **权限问题** - 某些系统目录可能无法访问（会跳过）
